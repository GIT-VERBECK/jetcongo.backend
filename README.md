# Règles de contribution sur github 

- Les règles suivantes doivent être strictement respectées par tous les membres du groupe afin d’assurer une collaboration propre et efficace.

## Politique des branches

** main

- Réservée au responsable du projet
- Aucun push direct autorisé
- Contient uniquement les versions stables et validées

** develop

- Branche par défaut du projet
- Aucun push direct autorisé
- Toute modification doit passer par une Pull Request

** feature/(nom_de_la_branche)

- Branches de travail personnelles
- Doivent être créées à partir de develop
- Une fonctionnalité ou correction par branche

### le nom de votre branche de travail devra commencer obligatoirement par : feature/nom_de_la_fonctionnalité

- Exemple : feature/login
- Exemple : feature/register 

** pour commencer le travail, cloner d'abord le projet sur votre pc en local puis switcher de branche 

```bash 
git checkout -b nom_de_la_branche

```
